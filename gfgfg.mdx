## Visão geral

Retorna as respostas das pesquisas de uma unidade. Cada registro inclui as respostas de todas as perguntas, os dados do contato, métricas de tempo e o AmpliScore.

```
POST https://api.amplifique.me/partners/cf/answers
```

<Callout kind="info">
  Use o parâmetro `only_finalized=complete` para retornar apenas pesquisas totalmente respondidas. Respostas parciais (contato abriu mas não concluiu) são retornadas com `finalized: false`.
</Callout>

---

## Autenticação

```bash
Authorization: YOUR_API_KEY
```

---

## Parâmetros de query

| Parâmetro | Tipo | Obrigatório | Descrição |
|---|---|---|---|
| `limit` | integer | Não | Quantidade de registros por página. Mín: `1`, Máx: `1000` |
| `page` | integer | Não | Página a retornar. Começa em `0` |
| `start` | date | Não | Data inicial (`dd/MM/yyyy`). Requer `end` |
| `end` | date | Não | Data final (`dd/MM/yyyy`). Requer `start` |
| `include_survey` | boolean | Não | Se `true`, retorna a estrutura completa da pesquisa junto de cada resposta |
| `only_finalized` | string | Não | Filtra por status de conclusão: `complete`, `incomplete` ou `all` |
| `survey` | string | Não | Filtra por pesquisa específica. Informe o ID da pesquisa. Ex: `67f578cccc4ab22df8543fcc` |

---

## Exemplo de requisição

```bash
curl -X POST "https://api.amplifique.me/partners/cf/answers?limit=1000&page=0&start=01/01/2025&end=31/01/2025&only_finalized=complete" \
  -H "Content-Type: application/json" \
  -H "Authorization: YOUR_API_KEY"
```

## Tipos de resposta por pergunta

O campo `answer` dentro de `answers` varia conforme o `type` da pergunta:

| Tipo (`type`) | Formato do `answer` | Exemplo |
|---|---|---|
| `nps` | integer (0–10) | `10` |
| `csat` | integer (1–5) | `4` |
| `ces` | integer (1–7) | `7` |
| `thumbs` | string (`up` ou `down`) | `"up"` |
| `likert` | integer | `5` |
| `text` | string | `"Gostei muito"` |
| `unique` | string (valor da opção) | `"agilidade"` |
| `multiple` | array de strings | `["agilidade", "qualidade"]` |
| `select` | string (valor da opção) | `"erros"` |
| `matrix` | object (coluna → valor) | `{ "atendimento": "Bom" }` |

## Erros comuns

| Código | Causa | O que fazer |
|---|---|---|
| `401` | API Key ausente ou inválida | Verifique o header `Authorization` |
| `400` | Parâmetros de data inválidos ou incompletos | Verifique se `start`+`end` foram enviados em par |

## Perguntas frequentes

<ExpandableGroup>
  <Expandable title="Como filtrar apenas respostas de uma pesquisa específica?">
    Use o parâmetro `survey` com o ID da pesquisa. Você encontra o ID da pesquisa na URL ao abrir a pesquisa na plataforma, ou no campo `_survey` das respostas já retornadas.
  </Expandable>

  <Expandable title="Qual a diferença entre answered, finalized e opened?">
    `opened: true` significa que o contato acessou o link da pesquisa. `answered: true` significa que respondeu ao menos uma pergunta. `finalized: true` significa que chegou ao final e submeteu a pesquisa. Para análise de dados, use `finalized: true` como critério principal.
  </Expandable>

  <Expandable title="O que é o AmpliScore (campo score)?">
    O AmpliScore é uma métrica calculada pela Amplifique.me que consolida as respostas de múltiplas perguntas em um único índice de 0 a 100. Quanto maior, melhor a avaliação geral do contato.
  </Expandable>

  <Expandable title="Como paginar corretamente para exportar todos os dados?">
    Faça a primeira requisição com `page=0` e `limit=1000`. Use o campo `pages` da resposta para saber o total de páginas e itere incrementando `page` até cobrir todas. O campo `total` informa o número total de registros disponíveis.
  </Expandable>
</ExpandableGroup>
